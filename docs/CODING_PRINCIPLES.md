# Engineering Principles

This is a personal MVP project.

Always prefer:

- Python 3.12
- uv package manager
- SQLite over PostgreSQL
- YAML over web dashboards
- GitHub Actions over cloud servers
- Email over mobile apps
- Single-user assumptions

Always:

- Use type hints
- Write pytest tests
- Use Ruff formatting
- Use structlog logging
- Keep prompts in separate markdown files
- Use repository pattern for database access

Avoid:

- Docker
- Kubernetes
- React
- Authentication systems
- Microservices
- Overengineering

The goal is to build a lightweight personal tool with minimal maintenance cost.
